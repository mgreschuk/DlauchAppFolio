## Color

The portal uses a neutral dark-surface theme appropriate for an internal operations tool.
This matches the "polished professional SaaS" intent from the project context.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f172a` (slate-950) | Page background, authenticated shell background |
| Secondary (30%) | `#1e293b` (slate-800) | Sidebar background, card surfaces, input backgrounds |
| Accent (10%) | `#3b82f6` (blue-500) | Reserved for: primary CTA button fill, active nav item indicator, AppFolio connectivity status "connected" dot |
| Destructive | `#ef4444` (red-500) | Log out confirmation only (see Copywriting Contract) |

Accent reserved for (exhaustive list — do not use accent on any unlisted element):
1. Primary CTA button fill ("Sign In" button on login page)
2. Active sidebar nav item left-border indicator (4px solid accent)
3. AppFolio connectivity status badge when status is "Connected"

Text on dominant: `#f8fafc` (slate-50) — primary text
Text on dominant (muted): `#94a3b8` (slate-400) — secondary/helper text
Border color: `#334155` (slate-700) — card borders, input borders, dividers

## Interaction States

All interactive elements must implement these states. No exceptions.

| State | Button (Primary) | Button (Text/Ghost) | Input | Nav Item |
|-------|-----------------|---------------------|-------|----------|
| Default | accent fill, white text | transparent, muted text | secondary bg, border | transparent |
| Hover | accent-600 fill (`#2563eb`) | slate-700 bg | border brightens to slate-500 | slate-700 bg |
| Focus | accent fill + 2px offset ring (accent color) | 2px offset ring | 2px offset ring (accent) | 2px offset ring |
| Active/Pressed | accent-700 fill (`#1d4ed8`) | slate-600 bg | — | — |
| Disabled | 40% opacity, cursor not-allowed | 40% opacity | 40% opacity | — |
| Loading | Spinner replaces label, button disabled | — | — | — |
| Active Nav | accent left border (4px), label semibold | — | — | left border accent |
| Error (Input) | — | — | border red-500, helper text red-400 | — |

Loading state applies to the Sign In button only in Phase 1 — show a spinner while
Auth.js processes the credential check. Button width must not change during loading
(set explicit min-width or use fixed-width layout).

## Accessibility Floor

- All form inputs have associated `<label>` elements (htmlFor linkage)
- Focus ring visible on every interactive element (never `outline: none` without replacement)
- Color is never the sole indicator of state — error inputs also show error text below the field
- Sidebar nav items meet 44px minimum touch target height (accessibility floor — not a spacing token)
- Login card is keyboard-navigable in tab order: Email → Password → Sign In button
