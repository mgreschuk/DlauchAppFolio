API Endpoints:
- GET /api/scopes — returns Scope[] (active only by default)
- GET /api/scopes?includeInactive=true — returns all Scope[]
- POST /api/scopes — body: { scopeName, category, vendor, workDescription } — returns created Scope (201) or error (400/409)
- GET /api/scopes/:id — returns single Scope or 404
- PUT /api/scopes/:id — body: { scopeName?, category?, vendor?, workDescription? } — returns updated Scope or 404/409
- PATCH /api/scopes/:id/toggle — returns toggled Scope with flipped isActive

Combobox implementation (per D-05):
Use shadcn Popover + Command components to build a combobox that:
- Shows a text input the user can type in freely
- As they type, a dropdown appears showing matching existing values from `existingCategories` or `existingVendors`
- User can select an existing value OR type a completely new one
- The typed/selected value is stored in form state

Implementation approach for the combobox:
```typescript
function Combobox({ value, onChange, options, placeholder }: {
  value: string;
  onChange: (val: string) => void;
  options: string[];
  placeholder: string;
}) {
  const [open, setOpen] = useState(false);
  const [inputValue, setInputValue] = useState(value);
  const filtered = options.filter(o => o.toLowerCase().includes(inputValue.toLowerCase()));

  return (
    <Popover open={open} onOpenChange={setOpen}>
      <PopoverTrigger asChild>
        <Button variant="outline" role="combobox" ...>
          {value || placeholder}
        </Button>
      </PopoverTrigger>
      <PopoverContent className="bg-[#1e293b] border-[#334155]">
        <Command>
          <CommandInput
            value={inputValue}
            onValueChange={(val) => { setInputValue(val); onChange(val); }}
            placeholder={placeholder}
            className="bg-[#0f172a] text-[#f8fafc]"
          />
          <CommandList>
            <CommandEmpty>No matches — will create new</CommandEmpty>
            <CommandGroup>
              {filtered.map(opt => (
                <CommandItem key={opt} onSelect={() => { onChange(opt); setInputValue(opt); setOpen(false); }}>
                  {opt}
                </CommandItem>
              ))}
            </CommandGroup>
          </CommandList>
        </Command>
      </PopoverContent>
    </Popover>
  );
}
```

If the shadcn Command component's API differs from the above, adapt accordingly — the key behavior is: type freely, see suggestions, select or keep typing.

Form state management:
- Use React useState for each field: scopeName, category, vendor, workDescription
- On mount / when editScope changes: if editScope is not null, pre-fill all fields from editScope. If null, clear all fields.
- Use a useEffect to sync editScope prop to form state.

Submit handler:
- If editScope is null (add mode): `fetch("/api/scopes", { method: "POST", headers: {"Content-Type": "application/json"}, body: JSON.stringify({ scopeName, category, vendor, workDescription }) })`
- If editScope is set (edit mode): `fetch(\`/api/scopes/${editScope.id}\`, { method: "PUT", headers: {"Content-Type": "application/json"}, body: JSON.stringify({ scopeName, category, vendor, workDescription }) })`
- On success (response.ok): call `onSuccess()` which triggers React Query invalidation in parent, call `onOpenChange(false)` to close modal, show `toast.success("Scope added")` or `toast.success("Scope updated")`
- On 409 response: show `toast.error("A scope with this name already exists")`
- On other errors: show `toast.error("Failed to save scope")`
- Disable submit button while request is in flight (track loading state)
