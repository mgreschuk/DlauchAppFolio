  id: uuid("id").primaryKey().defaultRandom(),

  vendor: varchar("vendor", { length: 255 }).notNull(),

});

Create three API route files. All routes use the established pattern: import `{ NextResponse }` from `"next/server"`, import `{ db }` from `"@/db"`, import `{ scopes }` from `"@/db/schema"`. Use drizzle-orm query builders (`eq`, `asc`, `desc` from `"drizzle-orm"`). Use Zod v4 for request body validation (already in package.json as `zod@^4.3.6`).

**IMPORTANT:** Read the Next.js 16 docs at `node_modules/next/dist/docs/` for the correct route handler signature. In Next.js 16, dynamic route params may be accessed differently than Next.js 14/15. Check the routing guide before implementing `[id]` routes. The `params` object in Next.js 15+ route handlers is a Promise that must be awaited: `const { id } = await params;`

**File 1: `src/app/api/scopes/route.ts`**

GET handler:
- Query `db.select().from(scopes)` ordered by `scopes.scopeName` ascending
- If query param `includeInactive=true` is present, return all entries
- Otherwise, filter with `.where(eq(scopes.isActive, true))`
- Access search params via `request.nextUrl.searchParams` (import `NextRequest` from `"next/server"`, use `request: NextRequest` as the function parameter)
- Return `NextResponse.json(rows)` with 200 status

POST handler:
- Parse JSON body, validate with Zod schema:
  ```typescript
  const createScopeSchema = z.object({
    scopeName: z.string().min(1).max(255),
    category: z.string().min(1).max(255),
    vendor: z.string().min(1).max(255),
    workDescription: z.string().min(1),
  });
  ```
- On validation failure, return `NextResponse.json({ error: "Validation failed", details: error.issues }, { status: 400 })`
- Insert into scopes table using `db.insert(scopes).values({...}).returning()`
- If insert fails due to unique constraint violation (catch error, check for `duplicate key` or `unique constraint` in error message), return `NextResponse.json({ error: "A scope with this name already exists" }, { status: 409 })`
- On success, return `NextResponse.json(newScope, { status: 201 })`

**File 2: `src/app/api/scopes/[id]/route.ts`**

GET handler:
- Extract `id` from params: `const { id } = await params;`
- Query `db.select().from(scopes).where(eq(scopes.id, id))`
- If no row found, return 404 with `{ error: "Scope not found" }`
- Return the scope object with 200

PUT handler:
- Extract `id` from params: `const { id } = await params;`
- Parse JSON body, validate with Zod schema:
  ```typescript
  const updateScopeSchema = z.object({
    scopeName: z.string().min(1).max(255).optional(),
    category: z.string().min(1).max(255).optional(),
    vendor: z.string().min(1).max(255).optional(),
    workDescription: z.string().min(1).optional(),
  });
  ```
- On validation failure, return 400 with error details
- Update using `db.update(scopes).set({ ...validated, updatedAt: new Date() }).where(eq(scopes.id, id)).returning()`
- If no row returned (0 rows updated), return 404 with `{ error: "Scope not found" }`
- If update fails due to unique constraint violation on scopeName, return 409 with `{ error: "A scope with this name already exists" }`
- On success, return the updated scope with 200

**File 3: `src/app/api/scopes/[id]/toggle/route.ts`**

PATCH handler:
- Extract `id` from params: `const { id } = await params;`
- First fetch current scope: `db.select().from(scopes).where(eq(scopes.id, id))`
- If not found, return 404
- Toggle: `db.update(scopes).set({ isActive: !currentScope.isActive, updatedAt: new Date() }).where(eq(scopes.id, id)).returning()`
- Return the updated scope with 200

All three files: wrap database operations in try/catch, return 500 with `{ error: "Internal server error" }` on unexpected failures. Do NOT log sensitive data.
